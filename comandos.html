Comandos de Rede

Diagnóstico e monitoramento em Windows e Linux

Ferramentas de linha de comando são essenciais para testar as camadas do Modelo OSI, especialmente as camadas de Rede e Transporte.

1. PING (Conectividade básica)

Verifica se um host está acessível usando mensagens ICMP (Camada 3).

Windows
ping google.com
Linux
ping -c 4 google.com

2. TRACERT / TRACEROUTE (Salto por salto)

Rastreia o caminho percorrido pelos pacotes até o destino.

Windows
tracert google.com
Linux
traceroute google.com

3. Configurações de Interface

Exibe detalhes dos adaptadores de rede e endereços IP.

Windows
ipconfig /all
Linux
ip addr show

4. NSLOOKUP (Consulta DNS)

Verifica a resolução de nomes de domínio (Camada 7).

Windows / Linux
nslookup google.com

5. NETSTAT (Conexões Ativas)

Lista todas as portas abertas e conexões TCP/UDP (Camada 4).

Windows
netstat -an
Linux
ss -tunlp

6. ARP (Tabela de Endereços MAC)

Exibe a correspondência entre IP e Endereço Físico (Camada 2).

Windows / Linux
arp -a

7. Identificar Endereço MAC

Obtém o endereço físico único da placa de rede.

Windows
getmac
Linux
ip link show

8. Tabela de Roteamento

Exibe as rotas que o sistema usa para enviar pacotes.

Windows
route print
Linux
ip route

9. Testar Porta Específica

Verifica se uma porta (ex: 80 para HTTP) está aberta em um servidor.

Windows
telnet google.com 80
Linux
nc -zv google.com 80

10. Informações de Domínio

Obtém detalhes do registro de um domínio ou registros DNS detalhados.

Windows (via PowerShell)
Resolve-DnsName google.com
Linux
dig google.com
